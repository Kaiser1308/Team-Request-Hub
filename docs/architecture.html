Team Request Hub — Kiến Trúc Hệ Thống

Tài liệu dành cho developer mới — hiểu toàn bộ hệ thống từ tổng quan đến chi tiết từng module.

1. Tổng Quan

Team Request Hub là công cụ quản lý yêu cầu nội bộ cho team FE, BE, Lead. Ngoài request workflow, hệ thống còn có kho file chung (Team Files) cho toàn team.

┌──────────────────────────────────────────────────────────┐ │ Browser / Next.js │ │ ┌──────────┐ ┌──────────────┐ ┌────────────────────┐ │ │ │ Login │ │ Dashboard │ │ Team Files │ │ │ │ Google │ │ Requests │ │ (File Explorer) │ │ │ │ OAuth │ │ Workflow │ │ │ │ │ └────┬─────┘ └──────┬───────┘ └─────────┬───────────┘ │ └───────┼────────────────┼───────────────────┼──────────────┘ │ │ │ ▼ ▼ ▼ ┌──────────────────────────────────────────────────────────┐ │ FastAPI Backend (:8000) │ │ ┌──────────┐ ┌──────────────┐ ┌────────────────────┐ │ │ │ Auth │ │ Requests │ │ Files │ │ │ │ Middle │ │ Service │ │ Service │ │ │ │ ware │ │ │ │ │ │ │ └────┬─────┘ └──────┬───────┘ └─────────┬───────────┘ │ └───────┼────────────────┼───────────────────┼──────────────┘ │ │ │ ▼ ▼ ▼ ┌───────────────────┐ ┌───────────────────────────────────┐ │ Supabase Auth │ │ Supabase PostgreSQL │ │ (JWT verify) │ │ users, internal_requests, │ └───────────────────┘ │ notifications, team_files, │ │ file_activity_logs, ... │ └───────────────────────────────────┘ │ ┌───────────┴───────────┐ ▼ ▼ ┌──────────────┐ ┌──────────────────┐ │ Supabase │ │ MinIO Storage │ │ Realtime │ │ (file bytes) │ │ (websocket) │ │ self-hosted │ └──────────────┘ └──────────────────┘
Nguyên tắc: Frontend chỉ dùng Supabase cho Auth/session. Tất cả business logic, permission, database query đều qua FastAPI. Frontend không bao giờ nhận MinIO credentials.

4. Luồng Xác Thực (Auth Flow)

1. User → /login → Google OAuth → Supabase Auth 2. Supabase → redirect /auth/callback → set session cookie 3. /auth/welcome → GET /users/me (FastAPI) ├── is_active=true → redirect vào dashboard └── is_active=false → hiện "ACCOUNT DISABLED", không cho vào 4. Mọi API call → Bearer JWT → FastAPI verify JWT → load user từ DB
Role mặc định: User mới luôn có role=fe, is_active=false. Lead phải duyệt (set is_active=true) trước khi user dùng được app.

5. Backend — Kiến Trúc Lớp (Layered Architecture)

routes/ ← HTTP request/response, parse input, gọi service │ ▼ services/ ← Business rules, permission checks, workflow, audit │ ├── repositories/ ← Supabase table access (SELECT, INSERT, UPDATE) ├── notification_module/ ← Notifications, Telegram delivery └── minio_storage/ ← MinIO presigned URLs, object copy/delete

5.1 Routes (routes/)

File mỏng, chỉ parse request → gọi service → trả response. Không chứa business logic.

# apps/api/app/routes/requests.py
@router.post("", response_model=InternalRequestOut, status_code=201)
async def create_request(
    payload: InternalRequestCreate,
    current_user: Annotated[CurrentUser, Depends(get_current_user)],
):
    require_active_current_user(current_user)
    return request_service.create_request(payload, current_user)

5.3 Repositories (repositories/)

Chỉ chứa code truy cập Supabase table. Không có business logic, không có permission check.

# apps/api/app/repositories/request_repository.py
REQUESTS_TABLE = "internal_requests"
REQUEST_LIST_COLUMNS = "id,title,description,tags,priority,status,..."

def get_request_or_404(request_id: str) -> dict:
    result = get_supabase_admin().table(REQUESTS_TABLE)\
        .select(REQUEST_DETAIL_COLUMNS).eq("id", request_id).limit(1).execute()
    if not result.data:
        raise NotFoundError("Request not found")
    return result.data[0]

7. Luồng Upload File (Presigned URL Flow)

Browser FastAPI MinIO │ │ │ │── POST /files/upload-url ─────►│ │ │ {name, size, content_type} │── validate user + size │ │ │── INSERT team_files │ │ │ (status=pending_upload) │ │ │── presigned_put_url() ──────►│ │◄── {file, upload_url, method}──│◄─────────────────────────────│ │ │ │ │── PUT upload_url (binary) ───────────────────────────────────►│ │◄── 200 OK ────────────────────────────────────────────────────│ │ │ │ │── POST /files/{id}/complete───►│ │ │ {size_bytes} │── UPDATE status=active │ │ │── log activity │ │◄── {file} ─────────────────────│ │
VibeSec: MinIO access key/secret chỉ tồn tại trong backend env. Presigned URL có TTL 300s. Frontend không bao giờ thấy credential thật.
