# Root FE Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a Next.js frontend skeleton in the repository root with the technologies and folder architecture from `tasks.md`.

**Architecture:** The repository root becomes the Next.js app. Application code lives under `src/`; route groups separate auth and dashboard pages; `lib/`, `providers/`, `types/`, and `components/` hold framework clients and shared skeleton code.

**Tech Stack:** Next.js 15 App Router, TypeScript, Tailwind CSS, Supabase SSR/client packages, TanStack Query v5, shadcn/ui with Slate theme.

- [ ] **Step 3: Create QueryProvider**

Use this content in `src/providers/query-provider.tsx`:

```tsx
"use client";

import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { ReactQueryDevtools } from "@tanstack/react-query-devtools";
import { useState } from "react";

export function QueryProvider({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(
    () =>
      new QueryClient({
        defaultOptions: {
          queries: {
            staleTime: 60 * 1000,
            refetchOnWindowFocus: false,
          },
        },
      }),
  );

  return (
    <QueryClientProvider client={queryClient}>
      {children}
      {process.env.NODE_ENV === "development" && (
        <ReactQueryDevtools initialIsOpen={false} />
      )}
    </QueryClientProvider>
  );
}
```
